Mental Model

Think of the dense path as the reference microscope and the production path as the real vLLM TPU serving route. For a small prompt, both paths receive the same parameters and token IDs. If the production path is correct, it should choose the same experts and produce nearly the same logits.

What To Scrutinize

Production attention correctness

Check that shared RPA gets full attention metadata, updates KV cache, applies the same exclusive self-attention post-processing as dense mode, and casts K/V to the cache dtype intentionally.

Debug mode remains simple

Dense attention should stay easy to read. It is a diagnostic/reference path, not the serving path to optimize.

Future TP/PP cost

The branch has explicit single-rank assumptions in the smoke harness and bespoke loading in the model implementation. That is acceptable for this diagnostic goal, but note the debt if pipeline or tensor parallel serving becomes urgent.
